# fitted Project Files — Upload Guide

These six files go in the **Files** section of your Claude.ai fitted Project (the right-side panel). Once uploaded, every chat in the Project can reference them without re-uploading.

## Upload order

Doesn't really matter, but if you want a sensible sequence:

1. `CLAUDE.md` — project ground truth
2. `fitted-design-system.md` — design reference
3. `fitted-stripe-conventions.md` — Stripe reference
4. `catalina-prompt-library.md` — your reusable prompts
5. `fitted-schema-TEMPLATE.md` — fill in actual schema, then upload (or skip and upload your existing schema doc)
6. `fitted-launch-plan-TEMPLATE.md` — fill in actual plan, then upload (or skip and upload your existing plan doc)

## What each file does

| File | Purpose | Generated by Claude or by you? |
|------|---------|-------------------------------|
| `CLAUDE.md` | Project ground truth — stack, pricing, hard rules. Reinforces what's in Instructions. | Claude (final, ready to upload) |
| `fitted-design-system.md` | Brand voice, palette, typography, component conventions, anti-patterns. | Claude (final, ready to upload) |
| `fitted-stripe-conventions.md` | SDK v22 gotchas, webhook debugging, RLS escape hatch rules. | Claude (final, ready to upload) |
| `catalina-prompt-library.md` | 18 reusable prompts organized by when to use them. | Claude (final, ready to upload) |
| `fitted-schema-TEMPLATE.md` | Supabase tables, RLS policies, migrations. | **TEMPLATE — you fill in actual SQL** |
| `fitted-launch-plan-TEMPLATE.md` | Phased launch plan with current status. | **TEMPLATE — you fill in actual phases** |

## A note on the templates

The schema and launch plan files are templates because if I'd generated them with invented content, Claude would reference my made-up version in every future chat — which is much worse than a clean template. Either fill them in yourself, or just upload your existing schema dump and launch plan docs directly to Files (probably easier).

## Don't forget

The **Instructions** field on the right side of the Project page is separate from Files. Paste the long instructions block I gave you earlier into the Instructions field via the `+` icon — that loads into every chat as a system prompt. Files load only when relevant or when you reference them.

## When to update these files

- `CLAUDE.md` — when you change stacks, pricing, or hard rules
- `fitted-design-system.md` — when you make a meaningful design system decision
- `fitted-stripe-conventions.md` — when you discover a new gotcha or change billing structure
- `catalina-prompt-library.md` — when you find a new prompt that earns its slot, or remove one you never use
- `fitted-schema.md` — every schema migration
- `fitted-launch-plan.md` — every time a phase advances or a decision gets made
